# 🎓 AI 기반 학점 관리 & 졸업 요건 분석 시스템

## 📖 프로젝트 개요
현대 대학생에게 있어 학점 관리와 졸업 요건 충족은 학업 과정에서 가장 중요한 과제 중 하나입니다.  
대부분의 대학 학사 시스템은 단순 성적 조회에 그쳐, 학생이 직접 복잡한 졸업 요건을 계산해야 하는 불편함이 존재합니다.  

**본 프로젝트의 목표**는 동국대학교 NDRIMS 성적표(PDF)를 기반으로 AI가 자동으로 학점을 분석하고,  
졸업 요건 충족 여부 및 맞춤형 과목 추천까지 제공하는 **웹 기반 학점 관리 서비스**를 개발하는 것입니다.  

---

## 🚀 개발 목표
- 📑 성적표 PDF 업로드만으로 학점 및 이수 현황 자동 분석
- ✅ 학과별 졸업 요건 매칭 및 부족 학점/미이수 과목 안내
- 🎯 개인 맞춤형 과목 추천 기능 (AI 기반)
- 🌐 직관적이고 사용자 친화적인 웹 서비스 구축

---

## 🛠️ 주요 기능
1. **PDF 업로드 및 성적표 분석**
   - OCR & NLP 기반 데이터 추출
2. **졸업 요건 매칭**
   - 전공/교양/필수/선택 학점 자동 분류
   - 부족 학점 계산 및 필수 과목 이수 여부 점검
3. **AI 기반 추천 시스템**
   - 협업 필터링 + 규칙 기반 추천 혼합 적용
   - 학습 패턴·선수 과목 고려한 맞춤형 제안
4. **웹 서비스**
   - 로그인 기반 개인 학점 관리 & 데이터 저장
   - 직관적인 UI/UX 제공

## 🌿 Git 브랜치 전략
본 프로젝트는 **GitHub Flow**를 기반으로 하되, 협업에 맞게 브랜치 전략을 다음과 같이 정의합니다:

- `main` : 배포용 브랜치 (항상 안정적인 버전 유지)
- `dev` : 개발 통합 브랜치 (기능 개발 완료 후 PR 머지 대상)
- `feat-이슈번호-기능명` : 개별 기능 개발 브랜치  
  예시) `feat-12-login`, `feat-27-pdf-parser`

### 💡 Workflow
1. 이슈 생성 → 이슈 번호 부여
2. 브랜치 생성: `feat-이슈번호-기능명`
3. 기능 개발 후 `dev` 브랜치로 Pull Request 생성
4. 코드 리뷰 및 머지
5. `main` 브랜치는 일정 단위로 `dev`에서 안정화된 버전만 머지

---

### 🔒 브랜치 보호 규칙 (GitHub Branch Protection)
- `main` 과 `dev` 브랜치는 **PR 필수, 직접 push 불가**
- **Force push 금지**
- **브랜치 삭제 금지**
- PR 머지 시 **최소 2명 리뷰 승인 필수**
- Merge 방식: Merge, Squash, Rebase 허용 (팀 상황에 맞게 선택)
